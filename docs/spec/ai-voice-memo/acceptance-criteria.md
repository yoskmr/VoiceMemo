### AC-101: ワンタップ録音開始 🔵

**関連**: [US-101](./user-stories.md#us-101-ワンタップ録音開始) / REQ-001

#### シナリオ1: 正常系 - ホーム画面からの録音開始

```gherkin
Given ユーザーがアプリのホーム画面を表示している
  And マイク権限が許可されている
When ユーザーが録音ボタンをタップする
Then 500ms以内に録音が開始される
  And 録音中を示すUIインジケータが表示される
  And 録音画面に遷移する
```

> **測定条件**: iPhone 15, iOS 17, ネットワーク非接続環境。p95基準（100回試行のうち95%以上が500ms以内）。

#### シナリオ2a: 異常系 - マイク権限未許可→許可時 🟡

```gherkin
Given ユーザーがアプリのホーム画面を表示している
  And マイク権限が未許可である
When ユーザーが録音ボタンをタップする
  And マイク権限を要求するシステムダイアログが表示される
  And ユーザーが「許可」を選択する
Then 録音が開始される
  And 録音中を示すUIインジケータが表示される
```

#### シナリオ2b: 異常系 - マイク権限未許可→拒否時 🟡

```gherkin
Given ユーザーがアプリのホーム画面を表示している
  And マイク権限が未許可である
When ユーザーが録音ボタンをタップする
  And マイク権限を要求するシステムダイアログが表示される
  And ユーザーが「許可しない」を選択する
Then 録音は開始されない
  And 「録音にはマイクの使用許可が必要です」というメッセージが表示される
  And 設定アプリへの誘導ボタンが表示される
```

### AC-102: リアルタイム文字起こし 🔵

**関連**: [US-102](./user-stories.md#us-102-リアルタイム文字起こし) / REQ-002

#### シナリオ1: 正常系 - 日本語音声のリアルタイム文字起こし

```gherkin
Given ユーザーが録音中である
  And オンデバイスSTTが有効である
When ユーザーが日本語で話す
Then 発話から2秒以内に文字起こし結果が画面に表示される
  And 文字起こしテキストがリアルタイムで更新される
  And 確定済みテキストと認識中テキストが視覚的に区別される
```

> **測定条件**: iPhone 15, iOS 17, ネットワーク非接続環境（オンデバイスSTT）。p95基準（100回試行のうち95%以上が2秒以内）。

#### シナリオ2: 正常系 - 無音区間の処理 🟡

```gherkin
Given ユーザーが録音中である
  And 5秒以上の無音が続いている
When 無音区間が検出される
Then 文字起こしの最後のセグメントが確定される
  And 無音区間は文字起こしテキストに含まれない
  And 録音は継続される
```

#### シナリオ3: 異常系 - 環境ノイズによる認識困難 🟡

```gherkin
Given ユーザーが録音中である
  And 環境ノイズが大きい
When STTが有意なテキストを生成できない
Then 「音声を認識できませんでした」というフィードバックが表示される
  And 録音データ自体は保存される
```

### AC-104: 録音の自動保存 🟡

**関連**: [US-104](./user-stories.md#us-104-録音の自動保存) / REQ-014

#### シナリオ1: 正常系 - バックグラウンド遷移時の自動保存

```gherkin
Given ユーザーが録音中である
When アプリがバックグラウンドに移行する
Then 録音データが自動的に中間保存される
  And バックグラウンドオーディオセッションにより録音が継続される
```

#### シナリオ2: 正常系 - 長時間バックグラウンド後の自動終了 🟡

```gherkin
Given ユーザーが録音中である
  And アプリがバックグラウンドに移行している
When バックグラウンド状態が10分以上継続する
Then 録音が自動的に停止・保存される
  And ユーザーにローカル通知で保存完了を知らせる
```

#### シナリオ3: 境界値 - ちょうど10分のバックグラウンド継続 🟡

```gherkin
Given ユーザーが録音中である
  And アプリがバックグラウンドに移行している
When バックグラウンド状態がちょうど10分（600秒）に達する
Then 録音が自動的に停止・保存される
  And ユーザーにローカル通知で保存完了を知らせる
  And 10分間の録音データが欠損なく保存されている
```

#### シナリオ4: 境界値 - 9分59秒のバックグラウンド継続 🟡

```gherkin
Given ユーザーが録音中である
  And アプリがバックグラウンドに移行している
When バックグラウンド状態が9分59秒に達する
Then 録音はまだ自動停止されず継続している
  And バックグラウンドオーディオセッションが維持されている
```

#### シナリオ5: 異常系 - アプリクラッシュ時のデータ保全 🟡

```gherkin
Given ユーザーが録音中である
  And 5秒間隔で中間保存が実行されている
When アプリが予期せずクラッシュする
Then 最後の中間保存時点までの録音データが保持される
  And 次回アプリ起動時に復旧した録音データが表示される
```

## エピック3: AI要約・タグ付け・感情分析（P3）

### AC-301: 録音完了後の自動AI要約 🔵

**関連**: [US-301](./user-stories.md#us-301-録音完了後の自動ai要約) / REQ-003

#### シナリオ1: 正常系 - 自動要約生成（オンライン・Proプラン）

```gherkin
Given ユーザーがProプランに加入している
  And ネットワーク接続がある
When 録音が完了する
Then クラウドLLMによりAI要約が自動生成される（STT精度基準: WER 10%以下）
  And 要約は録音テキストの下に表示される
  And 5分以内の録音に対し10秒以内に生成が完了する
```

> **測定条件**: iPhone 15, iOS 17, Wi-Fi接続環境（クラウドLLM利用時）。p95基準（100回試行のうち95%以上が10秒以内）。入力テキスト: 5分間の録音に相当する約1,500文字の日本語テキスト。
>
> **WER測定プロトコル**: 日本語音声テストセット（標準的な発話速度150-200文字/分、静音環境40dB以下、iPhone内蔵マイク使用）で100文以上を認識させ、文字単位の誤り率を算出。p95基準。

#### シナリオ2a: 正常系 - 無料プラン枠超過時のフォールバック 🟡

```gherkin
Given ユーザーが無料プランで利用している
  And 月のAI処理使用回数が10回に達している（上限到達済み）
When 録音が完了する
Then オンデバイスLLMにより基本的なAI要約が自動生成される
  And 要約はメモ詳細画面に表示される
  And 「今月のクラウドAI処理回数上限に達したため、オンデバイスで処理しました」と表示される
```

#### シナリオ2b: 正常系 - オフライン時のフォールバック 🟡

```gherkin
Given ユーザーがProプランに加入している
  And ネットワーク接続がない
When 録音が完了する
Then オンデバイスLLMにより基本的なAI要約が自動生成される
  And 要約はメモ詳細画面に表示される
  And 「オフラインのためオンデバイスで処理しました」という通知が表示される
  And ネットワーク接続回復後にクラウドLLMで再処理するオプションが提示される
```

#### シナリオ3: 正常系 - 手動再生成 🟡

