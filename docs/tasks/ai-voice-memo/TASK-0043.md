## タスク概要

メモ詳細画面に「つながるきおく」セクションを追加し、現在閲覧中のメモと関連性の高いメモを自動的にリンク表示する機能。Phase 1 では FTS5 + タグスコアリングによるキーワードベースの関連度算出、Phase 2 では CoreML NLEmbedding によるセマンティック類似度に移行する。Pro プラン限定機能。

extension RelatedMemoClient {
    static var liveValue: RelatedMemoClient {
        @Dependency(\.fts5IndexManager) var fts5IndexManager
        @Dependency(\.voiceMemoRepository) var repository

        return RelatedMemoClient(
            findRelated: { memoID, maxCount in
                guard let memo = try await repository.fetchMemo(memoID) else {
                    return []
                }
                return try await fts5IndexManager.findRelated(
                    memoID: memoID,
                    memoText: memo.transcription,
                    memoTags: memo.tags,
                    memoDate: memo.createdAt,
                    maxCount: maxCount
                )
            }
        )
    }
}

// Action に追加
case loadRelatedMemos
case relatedMemosLoaded([RelatedMemo])
case relatedMemosLoadFailed
case relatedMemoTapped(UUID)

// body に追加
case .loadRelatedMemos:
    guard state.isProUser else { return .none }
    state.isLoadingRelated = true
    return .run { [memoID = state.memo.id] send in
        let related = try await relatedMemoClient.findRelated(memoID, 5)
        await send(.relatedMemosLoaded(related))
    } catch: { _, send in
        await send(.relatedMemosLoadFailed)
    }

## テスト要件

### Phase 1: 正常系
- findRelated が FTS5 MATCH で関連メモを取得する
- タグスコアリングが一致率に基づく正しいスコアを返す
- 時間近接度が30日以内で高スコアになる
- 重み付きスコア合成が (FTS5*0.6 + タグ*0.3 + 時間*0.1) で算出される
- 閾値0.3未満のメモがフィルタされる
- 最大5件に制限される
- 自身のメモが結果から除外される

### Pro/Freeゲート
- Proユーザーは「つながるきおく」セクションが表示される
- 無料ユーザーはセクション自体が非表示
- Pro判定は SubscriptionClient を利用

### ナビゲーション
- relatedMemoTapped で対象メモの詳細画面に遷移する
- 遷移先でさらに関連メモが表示される（再帰的ナビゲーション）

### Phase 2: Embedding
- NLEmbedding でテキストからベクトルが生成される
- コサイン類似度が正しく算出される
- バックフィルが既存メモに対して段階的に実行される
- Embedding 未生成メモは FTS5 にフォールバックする

## 技術方針

### アーキテクチャ
- Phase 1: 既存 `FeatureMemo` + `InfraStorage` モジュール内に追加
- Phase 2: `InfraStorage` に Embedding 関連を追加
- 依存: Domain, InfraStorage, SharedUI, ComposableArchitecture

### スコアリングアルゴリズム（Phase 1）
- **FTS5 rank（重み60%）**: 全文検索のランクスコアを正規化（0.0〜1.0）
- **タグ一致率（重み30%）**: 共通タグ数 / max(対象タグ数, 候補タグ数)
- **時間近接度（重み10%）**: exp(-daysDiff / 30) で30日以内を優遇

### Phase 2 移行戦略
- Embedding 生成はバックグラウンドで段階的に実施
- 未生成メモは FTS5 にフォールバック
- Phase 2 完了後も FTS5 はフォールバックとして維持
