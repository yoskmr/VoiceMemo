## タスク概要

音声認識で生成された文字起こしテキストに対し、クラウドLLMを活用して「高精度仕上げ」を行う機能。句読点補正、口語→書き言葉変換、フィラー除去、段落整形などを実行し、読みやすいテキストに整える。元のテキストは保持し、いつでも切り替え可能。Pro プラン限定機能。

### Step 2: MemoDetailReducer/View の実装 + Backend エンドポイント（10h） :yellow_circle:

```swift
// FeatureMemo/MemoDetail/MemoDetailReducer.swift（既存拡張）
// State に追加
public var isPolishing: Bool = false
public var isPolished: Bool = false
public var showingOriginalText: Bool = false
public var originalTranscription: String?
public var polishError: PolishError?

public enum PolishError: Equatable {
    case network
    case timeout
    case apiError(String)
}

// Action に追加
case polishButtonTapped
case polishCompleted(PolishResult)
case polishFailed(PolishError)
case toggleOriginalText
case retryPolish

// body に追加
case .polishButtonTapped:
    guard state.isProUser else {
        state.destination = .proGate(.polish)
        return .none
    }
    guard !state.isPolished else { return .none }
    state.isPolishing = true
    state.polishError = nil
    return .run { [memo = state.memo] send in
        let request = PolishRequest(
            text: memo.transcription,
            customDictionary: customDictionaryClient.loadDictionary(),
            memoID: memo.id.uuidString
        )
        let result = try await backendProxyClient.polishText(request)
        await send(.polishCompleted(result))
    } catch: { error, send in
        if error is URLError {
            await send(.polishFailed(.network))
        } else {
            await send(.polishFailed(.apiError(error.localizedDescription)))
        }
    }

case let .polishCompleted(result):
    state.isPolishing = false
    state.isPolished = true
    state.originalTranscription = state.memo.transcription
    state.memo.transcription = result.polishedText
    // SwiftData 永続化 + FTS5 再インデックス
    return .merge(
        .run { [memo = state.memo] _ in
            try await voiceMemoRepository.updateMemo(memo)
        },
        .run { [memo = state.memo] _ in
            try await fts5IndexManager.reindex(memoID: memo.id, text: memo.transcription)
        }
    )

case .toggleOriginalText:
    state.showingOriginalText.toggle()
    return .none
```

```typescript
// Backend: POST /api/v1/ai/polish（Cloudflare Workers / Hono）
app.post('/api/v1/ai/polish', authMiddleware, proMiddleware, async (c) => {
    const { text, customDictionary, memoID } = await c.req.json()

    const systemPrompt = buildPolishPrompt(customDictionary)
    const response = await openai.chat.completions.create({
        model: 'gpt-4o-mini',
        messages: [
            { role: 'system', content: systemPrompt },
            { role: 'user', content: text }
        ],
        temperature: 0.3,  // 低めで安定した出力
        max_tokens: Math.min(text.length * 2, 4096),
    })

    return c.json({
        polishedText: response.choices[0].message.content,
        changeCount: countChanges(text, response.choices[0].message.content),
        processingTimeMs: Date.now() - startTime,
        modelVersion: 'gpt-4o-mini-2024-07-18',
        promptVersion: 'polish-v1.0.0',
    })
})

// InfraLLM/PromptTemplate+Polish.swift（新規）
extension PromptTemplate {
    static let polish = PromptTemplate(
        name: "polish",
        version: "1.0.0",
        system: """
        あなたは日本語の音声認識テキストを読みやすく整える校正アシスタントです。
        以下のルールに従ってテキストを仕上げてください:

        1. 句読点を適切に補正する（音声認識では句読点が欠落しやすい）
        2. フィラー（えーと、あのー、まぁ 等）を除去する
        3. 口語的な表現を自然な書き言葉に変換する（ただし元の意味を保つ）
        4. 適切な段落に分割する
        5. 明らかな認識誤りを文脈から推測して修正する
        6. 元の話者の意図や感情を損なわない
        7. 過度に形式的にしない（メモの自然さを保つ）

        {{#if customDictionary}}
        以下のカスタム辞書を参照してください:
        {{customDictionary}}
        {{/if}}

        仕上げたテキストのみを出力してください。説明は不要です。
        """,
        user: "{{text}}"
    )
}
```

```swift
// エラーハンドリング: リトライ指数バックオフ
// FeatureMemo/MemoDetail/MemoDetailReducer+Polish.swift（新規）
extension MemoDetailReducer {
    /// 指数バックオフリトライ（最大2回、2秒→4秒）
    func polishWithRetry(
        request: PolishRequest,
        maxRetries: Int = 2,
        initialDelay: Duration = .seconds(2)
    ) async throws -> PolishResult {
        var lastError: Error?
        for attempt in 0...maxRetries {
            do {
                return try await withTimeout(seconds: 30) {
                    try await backendProxyClient.polishText(request)
                }
            } catch {
                lastError = error
                if attempt < maxRetries {
                    let delay = initialDelay * pow(2, Double(attempt))
                    try await Task.sleep(for: delay)
                }
            }
        }
        throw lastError!
    }
}
```

## 技術方針

### アーキテクチャ
- 既存 `FeatureMemo` + `InfraNetwork` + `InfraStorage` モジュールに追加
- Backend は既存 Cloudflare Workers プロジェクトにエンドポイント追加
- 依存: Domain, InfraNetwork, InfraStorage, InfraLLM, SharedUI, ComposableArchitecture

### テキスト保持戦略
- 仕上げ前: transcription に元テキスト、originalTranscription = nil
- 仕上げ後: transcription に仕上げ済みテキスト、originalTranscription に元テキスト
- 元テキストは永久保持（削除不可）

### プロンプト設計方針
- temperature: 0.3（安定性重視）
- 元の意味・感情の保持を最優先
- 過度な形式化を避ける（メモの自然さを保つ）
- カスタム辞書による固有名詞・専門用語の補正

### エラーハンドリング戦略
- タイムアウト: 30秒（長文テキストの処理時間を考慮）
- リトライ: 指数バックオフ（2秒→4秒）、最大2回
- ネットワークエラー: ユーザーに明確なエラーメッセージ + 再試行ボタン
- 部分的成功: なし（全文仕上げのみ、部分仕上げは行わない）
