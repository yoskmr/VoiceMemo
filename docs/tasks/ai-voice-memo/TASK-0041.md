### Step 2: RAGパイプライン（6-8h） :large_blue_circle:

FTS5検索で質問テキストから関連メモを取得し、コンテキストを構築してBackend Proxyに送信。

```swift
// FeatureChat/RAGContextBuilder.swift
struct RAGContextBuilder {
    /// FTS5検索結果からLLMプロンプト用コンテキストを構築
    static func buildSystemPrompt(
        contexts: [RAGContext],
        memoTexts: [UUID: String]
    ) -> String {
        var prompt = """
        あなたはユーザーの音声メモ（きおく）に基づいて質問に回答するアシスタントです。
        以下のメモを参考にして、正確で簡潔な回答を日本語で提供してください。
        回答に使用したメモは [きおく1], [きおく2] のように参照番号で示してください。

        ---参照メモ---
        """

        for (index, context) in contexts.enumerated() {
            let memoText = memoTexts[context.id] ?? context.snippet
            prompt += """

            [きおく\(index + 1)] (\(context.createdAt.formatted()))
            タイトル: \(context.title)
            感情: \(context.emotion ?? "不明")
            タグ: \(context.tags.joined(separator: ", "))
            本文: \(memoText)
            """
        }

        return prompt
    }
}
```

### Step 3: Streaming Response（4-6h） :large_blue_circle:

SSE（Server-Sent Events）によるストリーミングレスポンスの受信。

```swift
// InfraNetwork/SSEStreamParser.swift
struct SSEStreamParser {
    /// URLSession.bytes からSSEイベントをパースしAsyncStreamに変換
    static func parse(
        bytes: URLSession.AsyncBytes
    ) -> AsyncStream<ChatStreamChunk> {
        AsyncStream { continuation in
            Task {
                var buffer = ""
                for try await byte in bytes {
                    let char = Character(UnicodeScalar(byte))
                    buffer.append(char)

                    if buffer.hasSuffix("\n\n") {
                        if let chunk = parseSSEEvent(buffer) {
                            continuation.yield(chunk)
                        }
                        buffer = ""
                    }
                }
                continuation.finish()
            }
        }
    }
}

public struct ChatStreamChunk: Equatable, Sendable {
    public let delta: String           // テキストの差分
    public let referencedMemoIDs: [UUID]?  // 最終チャンクのみ
    public let isFinished: Bool
}
```

## テスト要件

### 正常系
- sendButtonTapped で FTS5検索 → コンテキスト構築 → Backend送信 の一連フローが実行される
- ストリーミングチャンクが逐次 State.messages の最新メッセージに追記される
- streamCompleted で referencedMemoIDs が設定される
- 参照きおくカードタップで referencedMemoTapped(UUID) が送出される
- サジェスチョンタップで質問テキストが入力される

### RAGパイプライン
- FTS5検索が質問テキストで上位5-10件のメモを取得する
- コンテキスト構築でシステムプロンプトにメモテキスト+メタデータが含まれる
- コンテキスト長がLLMの上限を超えない（トークン制限）

### Pro/Freeゲート
- Freeユーザーの sendButtonTapped で proGatePresented が発生する
- Proユーザーは正常にチャットフローが実行される
- メモ3件未満でエンプティステートが表示される

### ストリーミング
- SSEパーサーが `data:` プレフィックスのイベントを正しくパースする
- `[DONE]` イベントで AsyncStream が完了する
- cancelStreamTapped でストリーミングが中断される
- ストリーミング中断後に isStreaming = false になる

### エラーハンドリング
- ネットワークエラー時に ChatError.network が設定される
- API エラーレスポンス時に ChatError.apiError が設定される
- タイムアウト時に ChatError.timeout が設定される
- エラー後に再送信が可能である

### UI
- バブルUIでユーザー/アシスタントのメッセージが区別表示される
- ストリーミング中はタイピングインジケーターが表示される
- 参照きおくカードに日付・タイトル・感情・タグが表示される
- 入力エリアが空の場合は送信ボタンが無効化される

## 技術方針

### アーキテクチャ
- 新モジュール `FeatureChat` として独立（FeatureMemo に追加しない）
- 依存: Domain, SharedUI, SharedUtil, ComposableArchitecture

### RAGパイプライン
1. ユーザー質問 → FTS5検索（FTS5IndexManagerClient）
2. 上位5-10件のmemoID取得 → VoiceMemoRepositoryClient.fetchMemosByIDs
3. メモテキスト+メタデータでコンテキスト構築
4. Backend Proxy経由でクラウドLLM（OpenAI API stream: true）
5. SSEチャンクをAsyncStreamで受信 → State.messages の最新メッセージに逐次追記

### プライバシー
- 既存のAI処理（processAI）と同じ方針: テキストのみクラウド送信、永続化しない
- DES-003 セクション7「プライバシー・データポリシー実装」準拠

### オンデバイスLLM
- RAGはクラウドLLM専用（Pro限定）。オンデバイスLLM（Phi-3-mini）はコンテキスト長が短く（~650トークン）RAG不向き
