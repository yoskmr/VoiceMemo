### Step 2: AIインサイト（10h） :yellow_circle:

感情分析データからAIによる振り返りインサイトを生成。Phase 1はテンプレートベース、Phase 2でクラウドLLMに移行。

```swift
// FeatureMemo/EmotionInsight/EmotionInsightReducer.swift（新規）
@Reducer
public struct EmotionInsightReducer {
    @ObservableState
    public struct State: Equatable {
        public var insight: EmotionInsight?
        public var isLoading: Bool = false
        public var period: InsightPeriod = .weekly
        public var cachedInsights: [InsightPeriod: EmotionInsight] = [:]

        public enum InsightPeriod: String, CaseIterable, Sendable {
            case weekly
            case monthly
            case quarterly
        }
    }

    public enum Action: Equatable {
        case onAppear
        case periodChanged(State.InsightPeriod)
        case insightGenerated(EmotionInsight)
        case insightFailed(String)
    }
}
```

```swift
// Domain/ValueObjects/EmotionInsight.swift（新規）
public struct EmotionInsight: Equatable, Sendable, Codable {
    public let period: String           // "2026-W14", "2026-03" 等
    public let summary: String          // インサイトテキスト
    public let dominantEmotion: String  // 最も多い感情
    public let memoCount: Int           // 対象メモ数
    public let generatedAt: Date
    public let version: String          // テンプレート/モデルバージョン
}
```

```swift
// InfraLLM/PromptTemplate+EmotionInsight.swift（既存拡張）
extension PromptTemplate {
    static let emotionInsight = PromptTemplate(
        name: "emotionInsight",

        system: """
        あなたはユーザーの感情パターンを分析し、優しく振り返りを提供するアシスタントです。
        以下の感情データから、ユーザーにとって有益な気づきを3点以内で提供してください。
        """,
        user: """
        期間: {{period}}
        メモ数: {{memoCount}}
        感情分布: {{emotionDistribution}}
        主要な感情の推移: {{emotionTrend}}
        """
    )
}
```

## テスト要件

### 正常系
- `.quarter` 期間で正しいデータ範囲が取得される
- AreaMark チャートが100%積み上げで描画される（データ合計が1.0）
- AIインサイトが正しい期間データで生成される
- 週次キャッシュが SwiftData に保存・読み込みされる
- 月次/四半期の振り返りビューが期間切替で正しく表示される

### Pro/Freeゲート
- Proユーザーは全期間・全機能にアクセス可能
- 無料ユーザーの trendIconTapped で Pro案内が表示される
- 無料ユーザーは最新3件分のみプレビュー表示される
- 3件より前のデータは鍵アイコンでマスクされる

### AIインサイト
- テンプレートベースのインサイトが正しい変数で生成される
- 週次キャッシュヒット時はAPI呼び出しが発生しない
- キャッシュミス時に新規生成される

### エンプティステート
- 感情分析済みメモ5件未満でエンプティステートが表示される
- エンプティステートに「メモを増やしましょう」的な案内が表示される

### エラーハンドリング
- インサイト生成失敗時にエラーメッセージが表示される
- エラー後に再試行が可能である
