## タスク概要

AVAudioEngineを用いたPCM 16kHz Mono録音エンジンを実装する。AAC/M4Aフォーマットでの圧縮保存、録音の開始/停止/一時停止/再開、リアルタイム音量レベルメーター機能を提供する。録音中一時ファイルにはNSFileProtectionCompleteUntilFirstUserAuthenticationを適用する。

### 2. AVAudioSession設定 :large_blue_circle:

NFR-001準拠: 録音開始500ms以内。アプリ起動時にセッションを事前設定。

```swift
let session = AVAudioSession.sharedInstance()
try session.setCategory(.playAndRecord, mode: .default, options: [.defaultToSpeaker, .allowBluetooth])
try session.setPreferredSampleRate(16000)
try session.setPreferredIOBufferDuration(0.01) // 10ms
try session.setActive(true)
```

### 3. AAC/M4A圧縮保存 :large_blue_circle:

NFR-006準拠: 1分あたり500KB以内（AAC 64kbps）。

```swift
let settings: [String: Any] = [
    AVFormatIDKey: kAudioFormatMPEG4AAC,
    AVSampleRateKey: 44100,
    AVNumberOfChannelsKey: 1,
    AVEncoderBitRateKey: 64000,  // 64kbps -> 約480KB/分
]
```

一時ファイルは `tmp/Recording/{UUID}_recording.m4a` に保存。

### 4. リアルタイム音量レベルメーター :large_blue_circle:

録音中のPCMバッファからRMS値を算出し、dBに変換して公開する。

```swift
/// 音量レベル（0.0 - 1.0 の正規化値）
var audioLevel: AsyncStream<Float> { /* PCMバッファからRMS算出 */ }

private func calculateRMS(buffer: AVAudioPCMBuffer) -> Float {
    guard let channelData = buffer.floatChannelData?[0] else { return 0 }
    let frameLength = Int(buffer.frameLength)
    var rms: Float = 0
    vDSP_measqv(channelData, 1, &rms, vDSP_Length(frameLength))
    return sqrt(rms)
}
```

### 5. 録音中一時ファイルの保護 :large_blue_circle:

統合仕様書セクション8.1準拠: 録音中一時ファイルは `NSFileProtectionCompleteUntilFirstUserAuthentication`。

```swift
try FileManager.default.setAttributes(
    [.protectionKey: FileProtectionType.completeUntilFirstUserAuthentication],
    ofItemAtPath: tempFileURL.path
)
```

### 6. バックグラウンド遷移時の自動保存 :large_blue_circle:

`UIApplication.willResignActiveNotification` を監視し、録音中にバックグラウンドに遷移した場合に自動的に中間保存を行う。

## テスト要件

### 正常系
- 録音開始で`isRecording`がtrueになること
- 録音停止で音声ファイル（M4A形式）が生成されること
- 一時停止/再開が正しく状態遷移すること
- AsyncStream<AVAudioPCMBuffer>にバッファが流れること
- 音量レベルメーターが0.0-1.0の範囲内の値を返すこと
- 生成された音声ファイルサイズがNFR-006基準を満たすこと

### 異常系
- マイク権限未許可時にエラーがスローされること
- AVAudioSession設定失敗時のエラーハンドリング
- 二重start防止（録音中にstartRecordingを呼んだ場合の挙動）
- ストレージ不足時の録音失敗ハンドリング

### バックグラウンド
- バックグラウンド遷移時に自動保存が実行されること
- 自動保存後にフォアグラウンド復帰で録音が継続可能なこと
