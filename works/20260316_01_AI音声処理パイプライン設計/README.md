# AI音声処理パイプライン設計

## 依頼内容
AI音声メモアプリの音声AI処理パイプライン設計書の作成

## 実施した作業内容
- 要件定義書（requirements.md）を精読し、AI処理関連の要件を抽出
- 音声AI処理パイプライン設計書（02-ai-pipeline.md）を作成

## 成果物
- `docs/spec/ai-voice-memo/design/02-ai-pipeline.md`

## 設計書の構成
1. **音声処理パイプライン全体図** - 録音からSTT、LLM処理、結果保存までのエンドツーエンドフロー（mermaid図）
2. **STTエンジン設計** - Apple SpeechAnalyzer（iOS 26+）/ WhisperKit（iOS 17-25）の実装設計、エンジン切替戦略
3. **LLM処理設計** - オンデバイス（Core ML）/ クラウド（GPT-4o mini）のハイブリッド構成、判定ロジック
4. **感情分析設計** - 8段階感情カテゴリ、JSON Schema、可視化データ設計
5. **AI処理キュー設計** - バックグラウンド処理、リトライ戦略、月5回制限管理、オフラインキューイング
6. **プロンプトテンプレート** - 統合プロンプト（要約+タグ+感情分析を1回で処理）、Few-shot例示
7. **パフォーマンス要件** - RTF目標、メモリ制約、メモリ管理戦略
8. **コスト試算** - GPT-4o miniのユーザーあたりコスト、損益分岐分析

## 得られた知見
- GPT-4o miniの低コスト（Proユーザー月約4.2円）により、LLMコストはほぼ無視できるレベル
- STTはWhisperKit smallモデル（600MB）がiPhone 15（6GB RAM）で最適なバランス
- STTとLLMの同時実行はメモリ制約上不可 → 排他制御で順次処理する設計
- 統合プロンプトにより1回のAPI呼び出しで要約+タグ+感情分析を処理し、コスト最適化
