# 競合アプリ UX・ユーザーストーリー分析と取り込み戦略

> **作成日**: 2026-03-24
> **目的**: 主要競合アプリのユーザーストーリー・ユーザー体験を分析し、MurMurNoteへの取り込み戦略を提案する
> **参照**: 既存調査6件（20260314〜20260315）+ 要件定義（requirements.md）+ ユーザー不満点調査

---

## 1. 競合アプリのユーザー体験パターン

### 1-1. Untold（音声日記 x 感情AI）

| 項目 | 内容 |
|:-----|:-----|
| **コアUX** | 録音 → Hume AIが音声から感情を自動検出 → 感情タイムラインで振り返り |
| **ユーザーストーリー** | 「自分の声のトーンから無意識の感情変化に気づきたい」 |
| **強み** | 完全無料、E2E暗号化、感情の可視化がバイラル要因 |
| **弱点** | 英語のみ、VC資金依存で持続性不明 |
| **UXの特徴** | 「声そのもの」を感情データとして扱う発想。テキスト化よりも音声の感情メタデータに価値を置く |

### 1-2. Voicenotes（第二の脳）

| 項目 | 内容 |
|:-----|:-----|
| **コアUX** | 声で話す → AIが構造化メモに変換 → タグ付けで知識ベース化 → AIに質問して引き出す |
| **ユーザーストーリー** | 「思いついたことを声で残し、後からAIに質問して引き出したい」 |
| **強み** | 100+言語対応、AIへの質問機能（RAG的体験）、15万+ユーザー |
| **弱点** | $10/月と高め |
| **UXの特徴** | 「蓄積 → 活用」のループ。使えば使うほどAIの回答精度が上がる体験設計 |

### 1-3. muute（日本市場ジャーナリング）

| 項目 | 内容 |
|:-----|:-----|
| **コアUX** | テキスト入力 → AI感情分析 → 週次レポートで振り返り |
| **ユーザーストーリー** | 「日記を書く習慣を通じて、自分の感情パターンを客観的に理解したい」 |
| **強み** | 100万DL、学校向け展開、認知行動療法的アプローチ |
| **弱点** | **音声入力なし**（テキスト入力のみ） |
| **UXの特徴** | 週次レポートによる「振り返りの習慣化」。継続率が高い設計 |

### 1-4. Awarefy（メンタルヘルスAI）

| 項目 | 内容 |
|:-----|:-----|
| **コアUX** | 感情記録 → 認知行動療法ベースのAI対話 → 200種セルフケア提案 |
| **ユーザーストーリー** | 「ストレスや不安を感じたとき、AIと対話して自分を客観視したい」 |
| **強み** | 80万DL、専門家監修のコンテンツ、法人向け展開 |
| **弱点** | 音声入力は限定的、ジャーナリングよりセラピー寄り |
| **UXの特徴** | 「記録 → 分析 → ケア提案」の3ステップ。能動的な改善行動を促す |

### 1-5. Just Press Record（シンプル録音）

| 項目 | 内容 |
|:-----|:-----|
| **コアUX** | Apple Watch含む全デバイスでワンタップ録音 → 自動文字起こし → iCloud同期 |
| **ユーザーストーリー** | 「いつでもどこでも、最速で声をメモに変えたい」 |
| **強み** | $4.99買い切り、Apple全デバイス統合、完全オンデバイス |
| **弱点** | AI機能なし（要約・整形なし） |
| **UXの特徴** | 「速さ」と「シンプルさ」に全振り。サブスク疲れユーザーに刺さる |

### 1-6. Granola（Bot不要会議録音）

| 項目 | 内容 |
|:-----|:-----|
| **コアUX** | 会議中にメモを取る → AIが録音とメモを統合して構造化ノート生成 |
| **ユーザーストーリー** | 「会議中は議論に集中し、ノートはAIに任せたい」 |
| **強み** | Bot不要（プライバシー配慮）、90-92%精度、急成長（評価額$250M） |
| **弱点** | $18/月と高額、会議特化でパーソナル用途に弱い |
| **UXの特徴** | 「手書きメモ + AI補完」のハイブリッド。人間の入力を尊重するUX |

### 1-7. シャべマル（日本発AI音声メモ）

| 項目 | 内容 |
|:-----|:-----|
| **コアUX** | 録音 → Whisper文字起こし → ChatGPTで要約・感情分析・カテゴリ分類 |
| **ユーザーストーリー** | 「話した内容をAIが自動で整理・分類してほしい」 |
| **強み** | 日本語対応、AI要約+感情分析、月額500円 |
| **弱点** | ユーザー規模極小、個人開発レベル |
| **UXの特徴** | MurMurNoteに最も近い設計思想。ただし品質・UX磨き込みが不十分 |

## 2. 競合UXから抽出したユーザー体験パターン

### パターンA: 感情可視化型（Untold / muute / Awarefy）

```
録音/入力 → 感情検出 → タイムライン表示 → 振り返り → 自己理解
```

**ユーザー価値**: 「自分では気づかない感情の変化を客観的に見える化する」
**習慣化メカニズム**: 週次レポート、感情推移グラフが「見たい」動機を生む

### パターンB: 知識蓄積型（Voicenotes / Granola）

```
録音 → AI構造化 → タグ付け・分類 → RAG検索 → 知識として再利用
```

**ユーザー価値**: 「蓄積した情報をAIが引き出してくれる第二の脳」
**習慣化メカニズム**: 使うほどAIの精度が上がる、データが増えるほど価値が増す

### パターンC: 即時変換型（Just Press Record / VOMO AI）

```
ワンタップ録音 → 即座に文字起こし → シンプルに保存
```

**ユーザー価値**: 「思いついた瞬間を逃さない最速の記録手段」
**習慣化メカニズム**: 摩擦ゼロの録音開始体験が「とりあえず録音する」習慣を作る

## 3. ユーザー不満点から見た競合UXの課題

### 3-1. 音声認識精度（最高深刻度）

| 不満 | 該当アプリ | MurMurNoteの対応状況 |
|:-----|:----------|:-------------------|
| 日本語で「存在しない単語を生成」レベルの誤認識 | Apple Voice Memos | Apple Speech + WhisperKit選択可能 + カスタム辞書（US-207） |
| 背景ノイズで精度60-70%に低下 | Otter.ai | オンデバイス処理で改善余地あり |
| 専門用語・固有名詞の誤認識が頻発 | 全般 | カスタム辞書（実装済み）で対応 |

### 3-2. テキスト変換後の活用しにくさ（高深刻度）

| 不満 | 該当アプリ | MurMurNoteの対応状況 |
|:-----|:----------|:-------------------|
| 段落分けなし・句読点なしの塊テキスト | 多数 | AI要約で構造化（Phase 3a） |
| 2時間の録音から特定箇所を見つけるのが「苦痛」 | 多数 | FTS5全文検索（実装済み）+ タグ自動付与（Phase 3a） |
| 録音しただけで後から見返さない | 全般 | **感情可視化・週次レポートで「見返す動機」を作る必要あり** |

### 3-3. プライバシー懸念（高深刻度）

| 不満 | 該当アプリ | MurMurNoteの対応状況 |
|:-----|:----------|:-------------------|
| 同意なき録音データの転送 | Otter.ai（2025年集団訴訟） | ローカル主保存（REQ-008） |
| 声紋（生体情報）の流出リスク | クラウドSTT全般 | オンデバイスSTT優先設計 |
| 45%がスマートスピーカーの音声データプライバシーを懸念 | 全般 | クラウド利用時はBackend Proxy経由 |

### 3-4. 料金・サブスク疲れ（高深刻度）

| 不満 | 該当アプリ | MurMurNoteの対応状況 |
|:-----|:----------|:-------------------|
| 月額$16.99〜$20は「保存するだけで高すぎる」 | Otter.ai / Notion | 月500円（$3.5）の低価格設定 |
| 無料→有料の機能制限が厳しい | 多数 | 無料でもSTT無制限 + AI月15回 |

## 4. MurMurNoteへの取り込み戦略

### 4-1. 優先度マトリクス

| 優先度 | 競合から学ぶUX | 対応するUS | 現状 | 提案アクション |
|:------:|:--------------|:----------|:-----|:-------------|
| **P1** | **感情の可視化**（Untold/muute） | US-303, US-304 | Phase 3b予定 | テキストベース簡易感情分析をPhase 3aに前倒し。オンデバイスLLMで3段階（ポジティブ/ネガティブ/ニュートラル）から開始 |
| **P2** | **蓄積メモへのAI質問**（Voicenotes） | **新規US-309** | 未定義 | RAG検索の新規ストーリー追加。「先週何を悩んでいた？」のような自然言語クエリ |
| **P3** | **週次振り返りレポート**（muute） | US-406に近い | Phase 4 | 感情分析 + トピック集計で「週次AIレビュー」として早期投入。継続率向上の鍵 |
| **P4** | **ワンタップ即録音の拡張**（Just Press Record） | US-101, US-405 | US-101実装済み | Apple Watch対応 + ホーム画面ウィジェットで「最速起動」をさらに磨く |
| **P5** | **構造化ノート自動生成**（Granola） | US-301に近い | Phase 3a | 要約に加えて「箇条書き・構造化」出力オプションを追加 |

### 4-2. 新規ユーザーストーリー候補

#### US-309: 蓄積メモへのAI質問（着想元: Voicenotes）

```
ユーザーとして、
蓄積したメモに対して自然言語で質問し、
過去の記録からAIが関連情報を引き出して回答してくれることで、
自分の「第二の脳」として活用したい。
```

**受け入れ基準案**:
- 「先週何について話した？」のような質問に、関連メモを引用して回答
- 直近30日分のメモをコンテキストとして使用
- オンデバイスLLM（Phase 3a）で基本動作、クラウドLLM（Phase 3b）で精度向上

#### US-310: 週次振り返りレポート（着想元: muute）

```
ユーザーとして、
週に一度、AIが感情・話題の傾向をまとめた振り返りレポートを自動生成してくれることで、
自分の思考や感情のパターンを客観的に把握したい。
```

**受け入れ基準案**:
- 毎週日曜日に自動生成（プッシュ通知でお知らせ）
- 話題TOP3、感情傾向、前週との比較を含む
- 最低3件以上のメモがある週のみ生成

#### US-311: 音声感情検出（着想元: Untold / Hume AI）

```
ユーザーとして、
声のトーンや話し方から、テキストでは表現しきれない感情ニュアンスを検出してもらうことで、
自分でも気づかなかった感情の変化を発見したい。
```

**受け入れ基準案**:
- 音声波形から感情ラベル（喜び/悲しみ/怒り/不安/平穏）を推定
- テキスト感情分析と音声感情分析の両方を表示
- Phase 3b以降（Hume AI or カスタムモデル、Backend Proxy必須）

#### US-312: Apple Watch録音（着想元: Just Press Record）

```
ユーザーとして、
Apple Watchから直接録音を開始し、
iPhoneと自動同期されることで、
iPhoneを取り出せない状況でも思いついたことを即座に記録したい。
```

**受け入れ基準案**:
- watchOS Companion Appでワンタップ録音
- Bluetooth/Wi-Fi経由でiPhoneに自動転送
- iPhone側で文字起こし・AI処理を実行

## 5. 差別化戦略サマリ

### MurMurNoteが勝てる3つのポジション

#### ポジション1: 「日本語特化 x 音声 x AI」の唯一のプレイヤー

| 比較軸 | Murmur | シャべマル | muute | Awarefy | **MurMurNote** |
|:-------|:------:|:--------:|:-----:|:-------:|:--------------:|
| 日本語特化 | 16言語の1つ | 対応 | N/A | N/A | **特化** |
| AI要約 | 対応 | 対応 | なし | なし | **対応** |
| 感情分析 | 対応 | 対応 | なし | 対応 | **計画中** |
| 音声入力 | 対応 | 対応 | **なし** | 限定的 | **対応** |
| ユーザー規模 | 11万 | 極小 | 100万 | 80万 | 0（初期） |

**戦略**: muute/Awarefy（100万DL級）が音声機能を追加する前に「日本語 x 音声 x AI」ポジションを確立する。

#### ポジション2: プライバシーファースト（オンデバイス完結）

| 比較軸 | Otter.ai | Notta | Untold | Just Press Record | **MurMurNote** |
|:-------|:--------:|:-----:|:------:|:-----------------:|:--------------:|
| オンデバイスSTT | なし | なし | なし | 対応 | **対応** |
| オンデバイスLLM | なし | なし | なし | なし | **Phase 3a** |
| ローカル保存 | なし | なし | 対応 | 対応 | **対応** |
| プライバシー訴訟リスク | 訴訟中 | なし | なし | なし | **最小化設計** |

**戦略**: Otter.ai訴訟（2025年）で高まるプライバシー懸念に応える。「あなたの声はあなたのデバイスの中に」をメッセージングの核にする。

#### ポジション3: 低価格フリーミアム（サブスク疲れ対策）

| 比較軸 | Otter.ai | Voicenotes | muute | **MurMurNote** |
|:-------|:--------:|:----------:|:-----:|:--------------:|
| 無料プランのAI | 制限あり | 制限あり | なし | **月15回** |
| 有料価格 | $8.33〜/月 | $10/月 | ¥750/月 | **¥500/月** |
| 無料でのSTT | 制限あり | 制限あり | N/A | **無制限** |

**戦略**: 無料でもSTT無制限 + AI月15回で十分な体験を提供し、Pro転換率を自然に高める。

## 6. 脅威と対策

### 6-1. Apple Journal + Siri統合（2026年春）— 最大の脅威

- **脅威**: 標準アプリの無料提供で「音声メモ + ジャーナリング」市場が侵食される
- **対策**: Apple がやらない深いAI機能（感情分析・タグ自動付与・RAG検索・週次レポート）で差別化。Apple Intelligence APIを活用しつつ、その上に独自価値を積む

### 6-2. muute/Awarefy の音声機能追加

- **脅威**: 100万DL級がいつ音声入力を搭載するかは不明
- **対策**: 早期の市場占有。「AI音声日記といえばMurMurNote」のブランド認知を先に構築

### 6-3. VC資金による大規模プレイヤー参入

- **脅威**: Granola（評価額$250M）のようなプレイヤーがAI日記市場に進出する可能性
- **対策**: 日本語特化という参入障壁。グローバルプレイヤーが日本語対応を後回しにする間にポジション確立
