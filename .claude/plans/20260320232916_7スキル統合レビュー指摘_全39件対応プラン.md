## Group B: Infra層 Concurrency修正（中難度・高優先度）

### B-1. AVAudioEngineRecorder: RTスレッドのロック改善（🔴#1）
**ファイル**: `ios/MurMurNoteModules/Sources/InfraSTT/Recording/AVAudioEngineRecorder.swift` L241-246
- Tapコールバック内の `lock.lock()`/`lock.unlock()` を既存の `withLock { }` ヘルパーに統一
- `withLock`内部のNSLockは残すが、RTスレッド内のロック保有時間を最小化
- 長期的にはos_unfair_lockへの移行を検討（TODOコメント）

### B-2. FTS5IndexManager: SQLiteコネクション永続化（🔴#2）
**ファイル**: `ios/MurMurNoteModules/Sources/InfraStorage/FTS5/FTS5IndexManager.swift`
- `private var db: OpaquePointer?` をクラスプロパティに昇格
- `init()` で `sqlite3_open()` を実行
- `deinit` で `sqlite3_close()` を実行
- `execute()` / `executeQuery()` / `executeLikeQuery()` から毎回のopen/closeを削除
- 既存の `lock` でスレッドセーフティを維持

### B-3. SwiftDataRepository: MainActor依存の軽減（🔴#3）
**ファイル**: `ios/MurMurNoteModules/Sources/InfraStorage/SwiftDataStore/SwiftDataVoiceMemoRepository.swift`
- SwiftData の `ModelContext` は MainActor 必須のため、完全な除去は不可能
- **軽減策**: `fetchAll()` で `fetchLimit` を指定しページネーション対応
- `save()` の Entity→Model 変換ロジックを `MainActor.run` の外に移動
- TODOコメント: iOS 18+ の `ModelContext(for:)` バックグラウンド対応を検討

### C-4. buildSections差分更新（🟡#18）
