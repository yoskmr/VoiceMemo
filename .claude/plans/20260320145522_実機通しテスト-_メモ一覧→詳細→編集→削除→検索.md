# 実機通しテスト: メモ一覧→詳細→編集→削除→検索

## Context

録音+STT+保存は実機で動作確認済み。しかし、その後のフロー（メモ一覧表示、詳細表示、テキスト編集、削除、検索）はコード接続済みだが実機未確認。
コード調査で以下の問題を発見したため、テスト前に修正が必要。

## 発見した問題

| # | 問題 | 影響 | 対応 |
|:--|:-----|:-----|:-----|
| 1 | `voiceMemoRepository.fetchMemos` が LiveDependencies に存在しない可能性 | メモ一覧が空になる | 確認・修正 |
| 2 | `fetchAllTags()` が空配列を返すスタブ | タグフィルターが動かない | 許容（Phase 3） |
| 3 | `AudioPlayerClient` がスタブ | 音声再生が動かない | 許容（Phase 3） |

## 実施ステップ

### Step 1: LiveDependencies のメモ一覧取得を確認
- `VoiceMemoRepositoryClient.liveValue` の `fetchMemos` クロージャが SwiftData 経由で正しくページネーション取得しているか確認
- 修正ファイル: `repository/MurMurNoteApp/LiveDependencies.swift`

### Step 2: Xcode MCP でビルド確認
- `BuildProject` → `GetBuildLog` でエラーがないことを確認

### Step 3: 実機で通しテスト
Xcode MCP の `RenderPreview` や実機操作で以下のフローを確認:

1. **録音 → 一覧**: 録音完了後、メモ一覧タブに自動切替されメモが表示されるか
2. **一覧 → 詳細**: メモタップでSheet表示、文字起こしテキストが表示されるか
3. **詳細 → 編集**: 編集ボタンタップで編集シート、テキスト変更→2秒デバウンス→自動保存
4. **削除**: 削除ボタン→確認ダイアログ→削除→一覧から消えるか
5. **検索**: 検索タブでFTS5検索、結果タップで詳細表示

### Step 4: 問題があれば修正

## 修正対象ファイル

- `repository/MurMurNoteApp/LiveDependencies.swift` — fetchMemos の SwiftData 接続確認
- `repository/MurMurNoteApp/MurMurNoteApp.swift` — AppReducer の画面遷移確認

## 検証方法

1. Xcode MCP `BuildProject` → `GetBuildLog` でビルド成功確認
2. 実機 or シミュレータで上記5フローを実行
3. Xcodeコンソールログで `[STT]` `[Reducer]` ログを確認
4. `swift test` で全352テストパスを維持
