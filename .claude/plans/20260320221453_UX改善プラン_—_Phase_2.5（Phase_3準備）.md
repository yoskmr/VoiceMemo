## Step 1: メモ詳細を Sheet → NavigationDestination (push遷移) に変更

### なぜ

- 設計書: Section 5.2 で `MemoList -->|"カードタップ"| MemoDetail` と push 遷移を指定
- push遷移ならスワイプバックで自然に戻れる

### UI仕様
```
メモ一覧画面
  ├─ メモカードタップ → push → メモ詳細画面（フルスクリーン）
  │    ├─ ナビバー左: ← 戻る（自動）

  │    └─ スワイプバックで一覧に戻る

  └─ 検索結果タップ → 同じく push → メモ詳細画面
```

## Step 3: 録音完了→メモ詳細に直接遷移

### なぜ
- 現状: 録音完了→メモ一覧に飛ばされ、自分のメモを探す必要がある
- 改善: 「メモを見る」→ 直接そのメモの詳細画面にpush遷移

### フロー
```
録音完了画面 →「メモを見る」タップ
  ↓
AppReducer: .recording(.navigateToMemoDetail(memoID))
  ↓
state.selectedTab = .memoList   ← タブ切替
.send(.memoList(.refreshRequested))  ← 一覧更新
.send(.memoList(.selectMemo(id: memoID)))  ← メモ詳細をpush
  ↓
MemoListReducer: state.selectedMemo = MemoDetailReducer.State(memoID: id)
  ↓
MemoListView: NavigationDestination → MemoDetailView
```

### タイミング問題への対策
- `refreshRequested` 完了前に `selectMemo` が届く可能性がある
- `MemoListReducer.State` に `pendingMemoID: UUID?` を追加
- `refreshRequested` 完了後に `pendingMemoID` があれば `selectedMemo` を設定

## Step 4: 感情分析をオプトイン（デフォルトOFF）

### なぜ
- 感情分析はクラウドLLM必須で月回数を消費する。ユーザーが意図せず回数を消費するのを防ぐ
- 日本文化では感情のラベリングに抵抗感がある人も多い
- オプトインにすることで「自分で選んだ」という納得感を生む

### UI仕様（設定画面に追加）
```
Section "AI処理" {
  ┌───────────────────────────────────┐
  │ 感情分析                    [OFF] │  ← Toggle, vmPrimary tint
  │ メモの内容をクラウドで分析し、    │     vmCaption1, vmTextTertiary
  │ 感情を判定します。AI処理回数を    │
  │ 1回消費します。                   │
  └───────────────────────────────────┘
}
```

## Step 5: 無料AI回数を月5回→月15回に引き上げ

### なぜ
- 月5回 = 毎日使うと5日で枯渇。AIの価値を体験する前に壁にぶつかる
- 月15回 = 毎日1回で2週間体験可能。「便利だけど制限がある」→ Pro転換の動機に
- 3タスク（要約+タグ+感情）が1カウントなので、15回 = 15メモ分

### AIProcessingQueueClient（DependencyKey）
```swift
struct AIProcessingQueueClient {
    var enqueueProcessing: @Sendable (UUID) async throws -> Void
    var observeStatus: @Sendable (UUID) -> AsyncStream<AIProcessingStatus>
    var cancelProcessing: @Sendable (UUID) async throws -> Void
}

enum AIProcessingStatus: Equatable, Sendable {
    case idle
    case queued
    case processing
    case completed
    case failed(String)
}
```

## 実装順序と依存関係

```
Step 1 (NavigationDestination) ← 基盤。Step 2,3 の前提
  ↓
Step 2 (録音完了画面) ← Step 3 の前提（navigateToMemoDetail アクション定義）
  ↓
Step 3 (録音完了→メモ詳細遷移) ← Step 1,2 に依存
  ↓
Step 4 (感情分析オプトイン) ← 独立、設計書+コード
Step 5 (月15回引き上げ) ← 独立、設計書のみ
Step 6 (AI処理バックグラウンド) ← Step 2 に軽く依存（完了画面のインジケーター）
```

Step 4 と Step 5 は独立なので Step 3 完了後に並行実行可能。
