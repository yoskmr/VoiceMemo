# iOSアプリにおけるUX方針

### 1. ユーザーの操作を必要以上に止めない
- 処理中であっても、関係のない操作まで止めない
- 待ち時間が発生する場合でも、可能な限り中断・キャンセルできるようにする
- ユーザーを強く拘束するUIは最小限にする
- 全画面で操作を止めるよりも、必要な範囲だけに影響を限定する
- 画面遷移や確認を増やしすぎず、できるだけ軽い操作で完結できる体験を優先する

### 2. 操作に対してすぐ反応を返す
- タップや選択の直後に、「操作が受け付けられた」と分かる反応を返す
- 結果を待たせるのではなく、まず安心感を返す
- 読み込み中でも、何が起きているか分かるようにする
- 画面は一気に完成を待つのではなく、表示できる部分から順に見せる
- 表示の揺れや突然のレイアウト変化を避け、見た目よりも理解しやすさを優先する

### 3. すぐに戻れる・やり直せるようにする
- ユーザーが安心して操作できるよう、戻る・閉じる・取り消す手段を前提に設計する
- 詳細を開いたあとも、元いた場所や文脈に戻りやすくする
- 誤操作しても回復しやすい体験を優先する
- 戻る負担が大きいUIは慎重に使い、より軽く戻れる構造を優先する

### 4. iOSらしい軽さと連続性を大切にする
- 1つの操作のたびに「構える」必要がある体験にしない
- 画面間の移動や状態変化は、できるだけ自然で連続的に感じられるようにする
- ユーザーが「今どこにいるか」「何をしたか」「次にどう戻れるか」を見失わないことを優先する
- 強い確認や大きな遮断よりも、流れを保ったまま安全に操作できる体験を目指す

### 5. 重要な体験に優先してコストをかける
- すべての画面で理想を徹底するのではなく、コア体験に優先して反映する
- 頻度が高い操作、失敗コストが高い操作、迷いやすい導線を優先的に改善する
- 周辺機能では、実装コストとのバランスを取りつつ、可能な範囲で同じ思想を保つ
